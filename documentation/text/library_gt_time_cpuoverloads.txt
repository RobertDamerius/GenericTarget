<h1>CPU Overloads</h1>
<h3>Library</h3>
<span style="font-family: Courier New">GenericTarget / Time / CPU Overloads</span>


<h3>Description</h3>
Get the number of <a href="timingscheduling.html#cpuoverload">CPU overloads</a> and lost ticks. During simulation this block has no effect.


<h2>Output Ports</h2>
<h3>numCPUOverloads</h3>
The number of CPU overloads that have been occurred. This value is incremented by one, if the timer interrupt event (with the base sampletime of the model) has been lost.
<br>
<b>Data Type:</b> <span style="font-family: Courier New">uint64</span>

<h3>numLostTicks</h3>
The number of lost ticks from the timer. This value is incremented by the number of lost timer interrupt events.
<br>
<b>Data Type:</b> <span style="font-family: Courier New">uint64</span>
